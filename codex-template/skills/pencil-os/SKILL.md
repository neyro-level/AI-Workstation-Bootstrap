# Pencil OS

Pencil OS - верхний навык-роутер для работы с Pencil.dev и `.pen`.

Задача skill - за 5 секунд понять режим задачи и направить работу в правильный специализированный skill.

## Принцип

Один запрос в Pencil обычно состоит из нескольких слоев:

1. Операционный слой: открыть, прочитать, изменить, экспортировать `.pen`.
2. Дизайн-слой: сделать современно, аккуратно, без AI-slop.
3. UX/B2B слой: Hero, CTA, форма, каталог, доверие, процесс, FAQ.
4. Design-system слой: tokens, variables, naming, lint, export/handoff.
5. Смысловой слой: H1, H2, CTA, буллиты, коммерческий текст.

Не пытайся решать все одним skill. Выбери минимальный набор.

## Всегда сначала

Если задача связана с реальным `.pen`:

1. Используй `pencil-mcp-operator`.
2. Не читай `.pen` shell-командами.
3. Начинай с `get_editor_state({ include_schema: true })`.

Если задача только концептуальная: можно сначала дать дизайн-решение без MCP, но явно назвать это концептом.

## Как работает старт дизайна

Пользователь может стартовать Pencil-задачу одним из трех способов:

1. Даёт готовый бриф, текст или ТЗ в чате.
2. Просит создать дизайн по проектным документам.
3. Открывает `.pen` и просит поправить/дорисовать макет.

В каждом случае не жди идеального ТЗ. Сам определи, чего хватает для первого шага.

### Если пользователь дал бриф или текст в чате

1. Определи тип результата: экран, страница, блок, компонент, дизайн-система, аудит.
2. Подключи нужные профильные skills.
3. Сформируй короткое дизайн-ТЗ: цель, аудитория, состав, CTA, ограничения, источники.
4. Если есть открытый `.pen`, предложи перейти к созданию skeleton.
5. Если `.pen` не открыт, дай концепт и попроси открыть/создать `.pen` для реализации.

### Если пользователь просит создать дизайн по проекту

1. Найди и прочитай `PASSPORT_PROJECTS.md`, `WORKLOG.md`, `DESIGN_SYSTEM.md`, `SITE_ARCHITECTURE*.md`, `PAGE_*.md`.
2. Если брифа страницы нет, создай/предложи `PAGE_*.md` до рисования.
3. Сформируй UX-роль и состав макета.
4. Затем переходи в Pencil MCP.

### Если пользователь просит править открытый `.pen`

1. Сразу подключи `pencil-mcp-operator`.
2. Получи editor state.
3. Если есть selection, работай с ним.
4. Если selection нет, прочитай верхний layout и найди нужный frame.
5. После правки обязательно проверь layout/screenshot.

## Маршрутизация

### Нужно работать с файлом Pencil

Подключай `pencil-mcp-operator`, если пользователь просит:

- открыть `.pen`;
- проверить canvas;
- прочитать selection;
- поправить блок в открытом Pencil;
- создать/изменить node/frame/text/image;
- экспортировать PNG/JPEG/WEBP/PDF;
- настроить или проверить Pencil MCP/CLI.

### Нужно сделать красиво и современно

Подключай `pencil-modern-ui`, если пользователь просит:

- сделать современный дизайн;
- усилить визуал;
- убрать AI-slop;
- нарисовать интерфейс;
- выбрать компоненты;
- использовать shadcn/Radix/Heroicons;
- улучшить типографику, spacing, карточки, кнопки, формы.

### Это коммерческий сайт или лендинг

Подключай `pencil-b2b-sites`, если задача про:

- B2B-сайт;
- недвижимость;
- ИЖС;
- инженерные услуги;
- Hero / первый экран;
- CTA;
- форму заявки;
- каталог/карточки;
- trust-блок;
- процесс;
- FAQ;
- посадочную страницу.

Если текст нужно усилить - дополнительно подключай `copywriter`.
Если UX нужно спроектировать - дополнительно подключай `design-ux`.

### Нужна дизайн-система

Подключай `pencil-design-system`, если задача про:

- `DESIGN_SYSTEM.md`;
- tokens / variables;
- цвета, шрифты, радиусы, spacing;
- дизайн-линт;
- consistency;
- export/handoff;
- перенос дизайн-системы в Pencil.

## Источники правды

Для клиентских проектов сначала ищи:

1. `PASSPORT_PROJECTS.md`
2. `WORKLOG.md`
3. `DESIGN_SYSTEM.md`
4. `SITE_ARCHITECTURE*.md`
5. `PAGE_*.md`
6. открытый `.pen` через Pencil MCP

`DESIGN_SYSTEM.md` важнее вкусовых рекомендаций. Не придумывай новые цвета, шрифты, радиусы и тени, если они уже определены проектом.

## Рабочий порядок

Для любой содержательной Pencil-задачи:

1. Определи режим: MCP / modern UI / B2B / design system / copywriting.
2. Назови пользователю, какие skills используешь и почему.
3. Собери минимальное дизайн-ТЗ из чата или проектных документов.
4. Если есть `.pen`, прочитай состояние через `pencil-mcp-operator`.
5. Если правка визуальная, примени `pencil-modern-ui`.
6. Если это коммерческий сайт, сверь с `pencil-b2b-sites`.
7. Если меняются токены или стилистика, сверь с `pencil-design-system`.
8. После правок проверь layout и screenshot.
9. Обнови проектные документы, если изменились смысл, структура или визуальная система.

## Что не делать

- Не читать `.pen` как обычный файл.
- Не смешивать Pencil CLI и VS Code extension без причины.
- Не превращать один запрос в создание новой дизайн-системы.
- Не использовать случайные градиенты, glassmorphism, glow, декоративные dashboard-карточки.
- Не делать дизайн без проверки screenshot/layout, если правка внесена в реальный `.pen`.
- Не копировать внешние skills дословно; использовать их как источники паттернов и адаптировать под текущий проект.
