# Pencil B2B Sites

Этот skill помогает рисовать в Pencil коммерческие страницы: B2B, недвижимость, ИЖС, услуги, локальные компании.

## Источники

Перед проектированием ищи:

1. `PASSPORT_PROJECTS.md`
2. `WORKLOG.md`
3. `SITE_ARCHITECTURE*.md`
4. `PAGE_*.md`
5. `DESIGN_SYSTEM.md`
6. текущий `.pen`

Если нет брифа страницы, создай или предложи создать `PAGE_*.md` рядом с материалами страницы.

## Связанные skills

- UX: `design-ux`.
- Текст: `copywriter`.
- Visual: `pencil-modern-ui`.
- Реальный `.pen`: `pencil-mcp-operator`.
- Реализация сайта: `your-site-build-skill` или `your-site-build-skill`.

## Принцип B2B-страницы

Каждый блок должен помогать пользователю:

- понять предложение;
- поверить;
- сравнить;
- снизить риск;
- выбрать;
- оставить заявку.

Не рисовать декоративные блоки без коммерческой роли.

## Hero

Hero должен сразу отвечать:

- что предлагается;
- для кого;
- какой результат;
- почему можно верить;
- какое действие сделать.

Визуал Hero должен быть связан с продуктом, объектом, процессом или результатом. Не использовать абстрактные dashboard-карточки для строительных/недвижимых проектов без причины.

## CTA

CTA должен быть конкретным:

- "Получить расчет";
- "Подобрать участок";
- "Записаться на консультацию";
- "Получить план работ";
- "Рассчитать стоимость".

Слабые CTA без контекста не использовать: "Подробнее", "Отправить", "Связаться".

## Формы

Форма должна быть короткой и понятной:

- 2-4 поля для первого касания;
- понятная кнопка;
- пояснение, что будет после заявки;
- privacy note, если требуется;
- error/focus states.

## Каталоги и карточки

Карточка должна помогать выбирать:

- ключевой параметр;
- цена/диапазон, если можно;
- площадь/срок/тип;
- изображение;
- CTA;
- статус/метка;
- 2-4 отличия.

Не делать карточку только красивой плиткой без критериев выбора.

## Trust

Доверие строится через:

- реальные факты;
- процесс;
- гарантии, если подтверждены;
- документы;
- опыт;
- фото/объекты;
- понятные условия.

Не выдумывать цифры, отзывы, гарантии, сроки, партнеров.

## Mobile

На mobile:

- CTA доступен рано;
- формы не превращаются в длинную анкету;
- карточки сканируются;
- изображения не съедают весь первый экран;
- текст не ломает кнопки и tabs.
